# ConvertOS (Bootloader + Minimal Kernel)

Минимальная учебная операционная система, реализующая загрузочный сектор и простое ядро с поддержкой базового взаимодействия с пользователем, обработкой клавиатуры и набором встроенных команд.

Проект состоит из загрузчика (ASM) и ядра (C/C++), работающих в среде x86 в реальном и защищённом режиме.

## Структура проекта
- `bootsect.asm` — загрузочный сектор (bootloader)
- `bootsect.bin` — бинарный образ загрузчика
- `kernel.cpp` — минимальное ядро ОС
- `kernel.bin` — бинарный образ ядра

## Возможности

### Загрузчик (bootloader)
- инициализация сегментов памяти
- работа с BIOS прерываниями (INT 0x10, INT 0x13, INT 0x16)
- вывод текста на экран
- переключение цветов консоли (gray, white, yellow, blue, red, green)
- выбор цвета через клавиши
- загрузка ядра с диска в память
- переключение в защищённый режим (protected mode)
- передача управления ядру

### Ядро (kernel)
- обработка клавиатурных прерываний (IRQ1)
- вывод текста в видеопамять (0xB8000)
- управление курсором
- базовый текстовый интерфейс командной строки
- прокрутка экрана
- встроенные команды

## Использование

После загрузки системы доступна консоль, в которой можно вводить команды.

### Поддерживаемые команды

- `info` — вывод информации о системе и текущем цвете текста  
- `clear` — очистка экрана  
- `shutdown` — завершение работы (через порт ввода-вывода)  
- `nsconv <число> <исходная_СС> <целевая_СС>` — перевод числа между системами счисления (2–36)  
- `szconv <размер> <единица1> <единица2>` — конвертация единиц хранения данных (b, k, m, g, t)  
- `posixtime <секунды>` — преобразование UNIX timestamp в дату и время  
- `wintime <число>` — преобразование Windows FILETIME в дату и время  

## Принцип работы

### Загрузчик
Загрузочный сектор выполняет начальную инициализацию системы, использует BIOS прерывания для:
- вывода интерфейса выбора цвета
- чтения клавиатуры
- загрузки секторов ядра с диска

После завершения настройки выполняется:
- переход в защищённый режим (Protected Mode)
- загрузка GDT
- передача управления ядру по фиксированному адресу

### Ядро
Ядро работает в 32-битном режиме и реализует:
- прямую запись в видеопамять VGA (0xB8000)
- обработку прерываний клавиатуры через IDT
- командный интерпретатор
- базовые математические и системные функции

## Технологии
- Assembly (AT&T syntax, GNU assembler)
- C / C++
- x86 BIOS interrupts (INT 10h, INT 13h, INT 16h)
- Protected Mode (GDT, IDT)
- VGA text mode (0xB8000)
- GCC (cross-compilation для ОС)

## Особенности реализации
- ручная работа с памятью и портами ввода-вывода
- реализация собственной IDT
- обработка прерываний клавиатуры без ОС
- переключение в защищённый режим без загрузчика ОС
- собственная реализация CLI внутри ядра

## Назначение
Проект предназначен для изучения:
- архитектуры x86
- принципов загрузки ОС
- работы с BIOS и аппаратными прерываниями
- низкоуровневого программирования
- построения минимального ядра операционной системы
